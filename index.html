Cloud Infrastructure & Secure Web Delivery

The project required the architecture of a hardened web server environment within a cloud ecosystem to mitigate modern web-based attack vectors. The objective was to deploy a functional LAMP stack while ensuring superior performance and security by default, utilizing Google Cloud Platform (GCP) as the primary provider.

My mandate involved the configuration of a scalable Compute Engine instance and the implementation of a multi-layer defense strategy. Key objectives included establishing Zero Trust network isolation, managing sensitive credentials through dedicated secrets services, and integrating continuous security scanning to identify and remediate package-level vulnerabilities.

  • Platform Provisioning & Hardening: Deployed an n2d-standard-2 instance (Ubuntu 25.04) enforcing Shielded VM features including Secure Boot, vTPM, and Integrity Monitoring.
  • Identity & Access Management: Implemented Google Cloud Identity-Aware Proxy (IAP) with mandatory Two-Factor Authentication (2FA) and OS Login, effectively eliminating the risk of unauthorized administrative access via credential theft.
  • Containerized Security: Orchestrated a MariaDB deployment within a private Docker bridge network, utilizing tmpfs mounts and read-only volume mapping to prevent secret persistence in image layers.
  • Edge Protection: Established a Cloudflare Tunnel for outbound-only connectivity, with WAF custom rules for geographic blocking and rate-limiting to mitigate automated DDoS.
  • Vulnerability Orchestration: Integrated the Trivy scanning engine to audit against CVE standards, successfully performing image slimming to remediate 100% of actionable vulnerabilities.

Secure-By-Design Esports Password Manager (Python)

Engineered a secure-by-design credential management PoC for the esports sector during my second year at the University of South Wales. Directed the implementation of a robust framework for password complexity validation, cryptographic storage, and Multi-Factor Authentication (MFA) to mitigate credential harvesting risks.

  • Developed a modular Python 3.9 codebase utilizing bcrypt for adaptive hashing and Fernet (AES) for symmetric encryption.
  • Engineered a TOTP-based MFA using pyotp, migrating from insecure SMTP protocols, and utilized a CLI to minimize the software's attack surface.

Validated system integrity through 22 rigorous test cases, successfully documenting specific vulnerabilities—including SQL injection vectors and logic errors—to inform future remediation strategies.

Network Infrastructure & Secure Endpoint Hardening

The project necessitated the architecture and implementation of a secure, resilient network infrastructure for Key Cyber. Operating within a four-person team (Group 8), the objective was to move from a conceptual star topology to a functional, audited prototype that could withstand modern cyber threats.

  • Platform Deployment: Provisioned a synchronized Windows Server 2022 instance to serve as the organizational backbone alongside a Windows 11 workstation.
  • Network Automation: Implemented KEA DHCP managing dynamic IP leasing and integrated devices natively under keycyber.co.uk with external 8.8.8.8 resolution.
  • Security Tooling: Installed and configured pfBlockerNG for DNS-based filtering and Snort as a NIDS, laying groundwork for Splunk SIEM architecture.
  • Gateway Hardening: Mandated HTTPS and HSTS protocols via FreeBSD pfSense routing and instantly rotated default administrative credentials using a multi-salt standard.
  • Vulnerability Management: Conducted comprehensive security auditing using Tenable Nessus Essentials, identifying and documenting critical configuration weaknesses.

Delivered a fully operational network prototype with verified stateful packet inspection and secure filtering, establishing a proactive strategic roadmap for "Least Privilege" operationalization.
